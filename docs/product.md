# Product Context

## Product

- 이름: 성덕순례
- 한 문장 설명: 드라마, 영화, 애니메이션의 촬영지와 배경 장소를 발견하고 AI로 성지순례 일정을 만들어 지도에서 확인하는 설치형 웹앱
- 주요 사용자: 좋아하는 작품을 따라 여행하고 덕질 경험을 확장하려는 일반 사용자

## Product value

성덕순례는 작품 감상에서 끝나지 않고 사용자가 실제 장소를 방문하는 경험으로 이어지게 한다.
촬영지 주변의 음식점과 방문 장소도 함께 추천해 사용자의 여행 편의를 높이고 지역 소상공인에게 새로운 방문 수요를 연결한다.

## Core journey

초기 제품의 핵심 사용자 흐름은 다음과 같다.

1. 사용자가 OTT 서비스와 유사한 콘텐츠 탐색 화면에서 드라마, 영화 또는 애니메이션을 발견한다.
2. 작품을 선택해 작품 정보와 주요 촬영지 또는 배경 장소를 확인한다.
3. 사용자가 명시적인 버튼을 눌러 AI 여행 일정 생성을 요청한다.
4. 생성된 일정에서 촬영지와 주변 음식점 또는 추천 장소를 확인한다.
5. 카카오 지도 등의 지도 서비스에서 일정의 위치와 이동 경로를 확인한다.

OTT 서비스는 정보 구조와 탐색 경험의 참고 대상이며, 특정 서비스의 디자인 자산이나 화면을 그대로 복제하지 않는다.

## Initial scope

- 모바일 우선의 설치형 웹앱(PWA)
- 작품 목록 및 콘텐츠 탐색
- 작품 상세와 주요 촬영지 목록
- 촬영지 주변 음식점 및 방문 장소 추천
- 버튼으로 시작하는 AI 일정 생성
- 생성된 일정을 지도에서 확인
- 백엔드 API와 연동 가능한 프론트엔드 경계

## Current constraints

- Next.js App Router와 React Server Components를 기본으로 사용한다.
- 제품은 모바일 우선의 단일 설치형 웹앱으로 제공한다.
- 별도 홍보 웹사이트와 iOS·Android 네이티브 앱은 초기 범위에 포함하지 않는다.
- 데스크톱 브라우저에서는 같은 웹앱을 반응형으로 제공하되 별도 제품 경험을 만들지 않는다.
- TypeScript strict mode를 유지한다.
- 백엔드는 별도 구현 후 API로 연동할 예정이다.
- 로그인 기능은 아직 계획되지 않았으므로 인증을 전제로 설계하지 않는다.
- 디자인과 세부 기능이 확정되기 전에는 UI, 전역 상태 관리, 폼 라이브러리를 추가하지 않는다.
- 현재 PWA 범위는 설치와 독립 실행에 필요한 Web App Manifest까지다.
- 오프라인 캐시, 푸시 알림, 백그라운드 동기화는 요구사항이 확정되기 전에는 추가하지 않는다.

## Open questions

- 작품과 촬영지 데이터의 출처, 사용 권한, 갱신 주기
- AI 일정 생성 API의 입력, 출력, 실패 및 재시도 계약
- 주변 장소 추천 기준과 광고·제휴·추천 결과의 표시 원칙
- 카카오 지도 API의 구체적인 제품과 키 관리 방식
- 일정 저장, 공유, 즐겨찾기 및 로그인 도입 여부
- 지원 브라우저와 접근성, 성능, 보안 및 운영 목표
- 첫 번째 릴리스에서 제외할 명시적 범위

## Local map setup

- 지도 화면은 브라우저에 노출 가능한 카카오 지도 JavaScript 키를 `NEXT_PUBLIC_KAKAO_MAP_APP_KEY`로 받는다.
- 로컬 도메인을 카카오 개발자 콘솔의 Web 플랫폼에 등록해야 실제 지도가 표시된다.
- 키가 없거나 SDK 로드가 실패하면 개발 및 자동화 검증을 위한 지도 미리보기를 표시한다.
- 상세 화면에서 촬영지를 선택하면 사용자와 촬영지의 거리를 먼저 확인한다. 3km 이내의 가까운 촬영지는 현재 위치를 출발지로 사용하고, 그보다 멀면 현재 위치 주변 20km의 지하철역, 기차역, 버스터미널 후보 중 목적지까지의 추정 이동시간이 가장 짧은 곳을 출발지로 선택한다.
- 교통시설 후보의 추정 이동시간은 현재 위치에서 후보까지의 접근 거리, 환승 비용, 후보에서 촬영지까지의 거리를 합산한다. 실제 도로·대중교통 소요시간 API가 연동되기 전까지는 직선거리와 평균 이동속도를 사용한 근사값이다.
- 선택한 출발지의 주소를 AI 일정 생성 API의 출발지로, 선택한 촬영지 주소를 목적지로 전달한다.
- 생성한 일정은 탭 단위 저장소에 보관하고 촬영지 목록에 `루트 생성` 상태를 표시한다. 같은 촬영지를 다시 선택했을 때 출발지 종류와 좌표가 기존 출발지 250m 이내로 같으면 저장 일정을 재사용하며 AI 일정 생성 API를 다시 호출하지 않는다.
- 저장 당시 출발지와 현재 출발지가 다르면 기존 일정을 재사용하지 않고 현재 출발지를 기준으로 AI 일정을 다시 생성한다.
- 카메라 화면에서 명장면 촬영 사진의 방문 기록 저장을 완료하면 탭 단위 저장소에 기록하고, 작품 상세의 해당 촬영지 오른쪽에 `방문완료` 상태를 표시한다. 같은 촬영지에 생성된 루트가 있어도 방문 완료 상태를 우선 표시한다.
- 출발지와 목적지 주소는 각 좌표를 기준으로 표준 도로명·지번 주소를 확인하며, 백엔드 주소 인식 오류에 한해 다음 표준 주소 후보로 자동 재시도한다.
- 지도와 일정 패널에는 전체 촬영지가 아니라 출발지, 선택한 촬영지, AI가 추천한 일정 장소만 표시한다.
- 일정은 출발지에서 시작해 주변 소상공인 추천 장소들을 응답 순서대로 방문하고 사용자가 선택한 촬영지를 마지막으로 방문한다.
- 일정 패널은 연속된 각 방문지 카드 사이에 두 장소의 좌표로 계산한 직선거리를 표시한다.
- 지도 화면의 `루트 따라가기`는 일정을 다시 생성하지 않고 현재 위치를 연속 추적하며 다음 방문지와 남은 직선거리를 안내한다.
- 방문지 45m 이내에 진입하면 다음 방문지로 자동 전환하고, 사용자가 지도를 조작하면 카메라 따라가기를 멈춘 뒤 `내 위치` 버튼으로 재개한다.
- 활성 일정 구간에서 120m 이상 벗어난 정확한 위치가 3회 연속 확인되면 현재 위치에서 다음 방문지까지 시각 경로를 다시 표시한다.
- 음성·진동 안내는 사용자가 직접 켠 경우에만 사용하며 화면 꺼짐 방지는 따라가기 중에만 요청한다.
- 현재 경로 선은 도로 보행 경로가 아닌 일정 장소 간 직선이므로 실제 턴바이턴 안내에는 별도 보행 경로 API가 필요하다.
- 현재 위치 권한이 거부되거나 위치·교통시설 조회가 실패하면 사용자에게 상태를 알리고 임의의 출발지를 서버로 전달하지 않는다.
- 브라우저 위치 정보는 `localhost` 또는 HTTPS 보안 환경에서만 요청한다.
