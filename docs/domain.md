# Domain Language

코드, API 계약, 작업 명세에서 아래 용어를 일관되게 사용한다. 백엔드 계약이 확정되면 실제 스키마에 맞춰 갱신한다.

## Work

사용자가 탐색하는 드라마, 영화 또는 애니메이션 작품이다.

예상 핵심 속성:

- 식별자
- 제목과 설명
- 작품 유형
- 포스터 또는 대표 이미지
- 공개 상태와 장르

## Filming location

작품에 등장한 실제 촬영지 또는 작품의 배경으로 소개할 수 있는 방문 장소다.

예상 핵심 속성:

- 식별자와 장소명
- 연결된 작품
- 주소와 좌표
- 작품 속 장면 또는 장소 설명
- 방문 시 유의사항

애니메이션처럼 실제 촬영이 없는 콘텐츠에도 같은 제품 흐름을 제공할 수 있도록 UI 문구에서는 필요에 따라 `작품 배경지` 또는 `성지`를 사용한다.

## Nearby place

촬영지 주변에서 함께 방문할 수 있는 음식점, 카페, 상점 또는 관광 장소다.

추천 이유와 출처를 사용자에게 설명할 수 있어야 하며, 광고나 제휴가 포함되면 명확히 구분해야 한다.

## Itinerary

사용자의 요청에 따라 AI가 생성한 여행 일정이다. 하나 이상의 날짜와 순서가 있는 방문 지점으로 구성된다.

AI 생성 결과는 확정 예약이나 절대적인 사실로 표현하지 않으며, 영업시간과 이동시간처럼 변할 수 있는 정보의 확인 방법을 제공해야 한다.

## Itinerary stop

일정 안의 개별 방문 지점이다. 촬영지 또는 주변 장소를 참조하며 방문 순서, 예상 시각, 체류시간 및 이동 정보를 포함할 수 있다.
