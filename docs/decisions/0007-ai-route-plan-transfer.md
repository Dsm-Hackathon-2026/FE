# 0007. AI route plan transfer

- 상태: 승인됨
- 날짜: 2026-07-14

## Context

AI 추천 일정은 사용자가 상세 화면에서 촬영지를 선택할 때 생성되지만 결과는 별도 지도 라우트에서 표시한다. 추천 응답은 장소 주소를 제공하되 지도 핀에 필요한 좌표는 제공하지 않으며, 전체 추천 결과를 URL에 포함하면 길이와 개인정보 노출 문제가 생긴다.

## Decision

- 상세 화면에서 촬영지가 현재 위치 3km 이내면 현재 위치에서 바로 시작하고, 그보다 멀면 주변 20km의 역·터미널 후보 중 목적지까지의 추정 이동시간이 가장 짧은 출발지를 찾아 AI 추천 API를 호출한다.
- 추정 이동시간은 `현재 위치 → 후보` 접근 거리와 환승 비용, `후보 → 목적지` 거리를 평균 이동속도로 환산해 비교한다. 실제 도로·대중교통 소요시간 API가 도입되면 이 휴리스틱을 해당 API 결과로 교체한다.
- 출발지와 목적지 좌표를 카카오 역지오코딩해 표준 도로명·지번 주소 후보를 만든다.
- 추천 API가 주소 인식 오류로 400을 반환하면 아직 시도하지 않은 표준 주소 후보 조합으로 제한적으로 재시도한다. 다른 400 또는 서버 오류는 자동 재시도하지 않는다.
- 추천 장소 주소는 카카오 주소 검색으로 좌표화하며 좌표를 찾지 못한 장소는 지도 일정에서 제외한다.
- 출발지, 선택한 촬영지, 좌표화된 AI 추천 장소만 일정 스냅샷에 포함한다.
- 지도 일정은 `출발지 → AI 추천 소상공인 장소 → 사용자가 선택한 촬영지` 순서로 조립하며 선택한 촬영지는 항상 마지막에 한 번만 포함한다.
- 백엔드 타임라인의 출발지·목적지 항목은 표준화한 주소, 25m 이내 좌표, 응답 메타 장소명을 기준으로 제외하고 추천 장소의 응답 순서는 유지한다.
- 백엔드가 제공하는 추천 장소 좌표가 있으면 우선 사용하고, 좌표가 없는 항목만 주소 검색으로 좌표화한다.
- 일정 스냅샷은 버전과 콘텐츠·목적지 식별자를 포함해 탭 단위 `sessionStorage`에 저장하고 지도 URL에는 목적지 식별자만 전달한다.
- 상세 화면은 저장된 일정이 있는 촬영지에 `루트 생성` 상태를 표시한다. 재선택 시 현재 출발지를 먼저 계산하고, 출발지 종류가 같으며 저장 좌표와 250m 이내이면 스냅샷을 재사용해 AI 추천 API 호출을 생략한다.
- 현재 출발지가 저장 출발지와 250m보다 멀거나 출발지 종류가 바뀌면 현재 출발지 기준으로 일정을 다시 생성해 스냅샷을 교체한다.
- 지도 화면은 저장된 스냅샷만 사용하며 전체 촬영지 API 결과로 핀을 만들지 않는다.

## Consequences

- 같은 탭에서 지도 화면을 새로고침해도 생성한 일정이 유지된다.
- 새 탭이나 세션 만료 후 지도 URL에 직접 접근하면 상세 화면에서 목적지를 다시 선택해야 한다.
- 카카오 지도 키, 위치 권한 또는 장소 검색이 준비되지 않으면 상세 화면에서 일정을 생성할 수 없다.
- 카카오 역지오코딩으로 얻은 도로명·지번 주소를 모두 백엔드가 인식하지 못하면 사용자에게 다른 장소를 선택하도록 안내한다.
- 일정 순서가 변경됐으므로 이전 버전의 탭 저장 일정은 무효화하고 상세 화면에서 다시 생성하도록 한다.
