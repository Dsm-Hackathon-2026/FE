# 0002. Core journey and integration boundaries

- 상태: 승인됨
- 날짜: 2026-07-13

## Context

성덕순례는 작품 탐색, 촬영지 발견, AI 일정 생성, 지도 확인을 하나의 사용자 흐름으로 제공해야 한다.
디자인, 백엔드 API, 지도 제품과 인증 요구사항은 아직 확정되지 않았다.

## Decision

- 초기 핵심 흐름을 `작품 탐색 → 작품 상세/촬영지 → AI 일정 생성 → 지도 확인`으로 정의한다.
- 콘텐츠 탐색 경험은 OTT 서비스를 참고하되 특정 서비스의 시각 자산과 화면을 복제하지 않는다.
- 백엔드와 지도 제공자는 명시적인 어댑터 경계 뒤에 둔다.
- AI 일정은 사용자의 명시적인 버튼 동작으로만 생성한다.
- 인증은 초기 핵심 흐름의 전제조건으로 두지 않는다.

## Consequences

- 세부 디자인이 바뀌어도 핵심 E2E 흐름을 안정적인 인수 조건으로 사용할 수 있다.
- 백엔드와 지도 API 계약이 확정되기 전 mock 또는 fixture 기반 개발이 가능하다.
- 일정 저장이나 개인화 기능에는 추후 인증 및 데이터 보존에 관한 별도 결정이 필요하다.
