# 0008. Route-following companion mode

- 상태: 승인됨
- 날짜: 2026-07-14

## Context

생성된 AI 일정은 여러 장소의 순서를 제공하지만 기존 지도 화면은 정적인 핀과 선만 보여 줬다. 사용자가 실제 여행 중 다음 장소를 놓치지 않으려면 현재 위치를 중심으로 한 안내가 필요하다. 다만 현재 프론트엔드가 가진 경로 데이터는 도로 형상이 아니라 일정 좌표 사이의 직선이다.

## Decision

- 브라우저 `watchPosition`으로 현재 위치를 연속 추적한다.
- 다음 방문지까지의 직선거리와 활성 구간을 표시하고 45m 반경 진입 시 다음 방문지로 전환한다.
- 사용자가 지도를 조작하면 카메라 추적만 일시정지하고 위치 수신은 유지한다.
- 위치 정확도 100m 이내인 측정에서 활성 직선 구간으로부터 120m 이상 떨어진 상태가 3회 연속 확인되면 이탈로 판정한다.
- 이탈 시 현재 위치에서 다음 방문지까지 활성 선을 다시 그리되, 이를 실제 보행 경로 재탐색으로 표현하지 않는다.
- 음성 및 진동은 기본 비활성화하고 사용자 선택 후에만 안내한다.
- Screen Wake Lock은 따라가기 중에만 요청하며 미지원·거부 시 핵심 기능을 계속 제공한다.

## Consequences

- 브라우저만으로 일정 진행과 기본 이탈 안내를 제공할 수 있다.
- GPS 정확도와 브라우저 백그라운드 정책에 따라 위치 갱신 품질이 달라질 수 있다.
- 실제 도로 기반 선형, 회전 안내, 예상 도보 시간, 정식 재탐색은 보행 경로 API 도입 후 별도 확장해야 한다.
