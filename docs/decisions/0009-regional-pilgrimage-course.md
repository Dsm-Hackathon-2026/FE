# 0009. Regional pilgrimage course

- 상태: 승인됨
- 날짜: 2026-07-15

## Context

작품 상세의 개별 촬영지 일정 외에 같은 지역의 여러 명장면 장소와 주변 맛집·카페를 하루 코스로 둘러보는 시연이 필요하다. 백엔드는 출발지와 여러 촬영지 ID를 받는 `/routes/pilgrimage`를 제공하지만, 응답 타임라인에는 촬영지 ID와 장소 유형이 포함되지 않는다.

## Decision

- 촬영지 주소의 첫 광역 행정구역을 기준으로 지역을 묶고 3곳 이상인 지역만 코스 카드로 표시한다.
- 한 지역 코스는 촬영지 3곳을 사용한다. 도깨비 서울 시연은 백엔드 호출이 검증된 촬영지 ID 24, 25, 26을 고정 사용한다.
- 지역 코스 요청은 현재 위치를 기준으로 선택한 출발지 주소, 현재 시각, 촬영지 ID 배열을 `/routes/pilgrimage`에 전달한다.
- 실응답의 snake_case 메타 필드와 nullable `destination`, `destinations` 배열을 별도 API 파서에서 검증한다.
- 응답 타임라인의 주소가 원본 촬영지 주소와 같거나 좌표가 100m 이내이면 해당 항목을 촬영지로 연결한다. 지도와 방문 인증에는 원본 촬영지 이름, 이미지, 좌표와 `spotId`를 사용한다.
- 주변 장소는 동일한 정규화 주소가 반복되면 한 번만 표시한다.
- 일정은 `region-<region>` 코스 ID로 `sessionStorage`에 저장한다. 촬영지별 카메라 화면은 코스 ID가 아니라 일정 stop의 `spotId`로 방문을 인증한다.
- 지도 최초 선택 장소는 일정의 마지막 촬영지로 정한다.

## Consequences

- 기존 지도, 일정 패널, 따라가기와 방문 인증 화면을 다중 촬영지 코스에서도 재사용할 수 있다.
- 백엔드가 타임라인에 `spotId`와 장소 유형을 추가하면 주소·좌표 대조 로직을 명시적 필드 기반으로 교체해야 한다.
- 백엔드가 자체 저장 촬영지 주소를 인식하지 못하면 지역 코스 생성이 실패하므로 시연 외 촬영지 조합은 별도 검증이 필요하다.
- 현재 API에는 종료 시각과 일수 제한이 없어 촬영지 수를 3곳으로 제한한다.
