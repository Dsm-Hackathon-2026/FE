# 0010. 클라이언트 locale 상태와 타입 안전 번역 사전

- 상태: 승인됨
- 날짜: 2026-07-15

## Context

서비스의 한국어 UI에 영어 지원과 런타임 언어 전환이 필요하다. 현재 앱은 인증이나 사용자 설정 저장소가 없고, 모든 경로를 locale별로 복제하는 것은 초기 제품 범위에 비해 라우팅과 배포 복잡도를 크게 늘린다.

## Decision

- 초기 지원 언어는 한국어(`ko`)와 영어(`en`)다.
- 앱 루트의 `I18nProvider`가 현재 locale과 번역 함수를 제공한다.
- 번역 키는 한국어 사전에서 타입을 파생하고 영어 사전이 모든 키를 구현하도록 TypeScript로 검사한다.
- 사용자가 선택한 locale은 `localStorage`에 저장하고 이후 경로 이동과 재방문에 적용한다.
- locale이 바뀌면 문서의 `html.lang`, 화면 문구, 접근성 레이블, 지도 음성 안내 언어를 함께 갱신한다.
- 백엔드가 제공하는 작품명, 장소명, 주소, 설명은 원본 콘텐츠이므로 번역하지 않는다. 향후 백엔드가 locale별 콘텐츠를 제공하면 API 계약에 locale을 추가한다.

## Consequences

- 새 사용자 표시 문구는 `src/i18n/dictionaries.ts`의 두 사전에 함께 추가해야 한다.
- 초기 서버 렌더링 언어는 한국어이며, 저장된 영어 선택은 hydration 후 적용된다.
- 검색엔진용 locale별 URL과 locale별 Web App Manifest가 필요해지면 별도 결정으로 서버 locale 라우팅을 도입한다.
