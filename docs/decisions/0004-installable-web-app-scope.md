# 0004. Installable web app scope

- 상태: 승인됨
- 날짜: 2026-07-13

## Context

초기 기반은 웹과 모바일 화면을 하나의 코드베이스로 제공하는 방향이었지만, 제품 전달 범위를 별도 웹사이트와 앱으로 나누면 디자인, 배포 및 운영 경계가 불필요하게 커진다.
초기 제품은 앱 중심 경험에 집중하되 별도의 네이티브 코드베이스를 만들지 않아야 한다.

## Decision

- 제품을 모바일 우선의 단일 설치형 웹앱(PWA)으로 제공한다.
- Web App Manifest와 `standalone` 표시 모드를 설치 및 독립 실행의 기본 경계로 사용한다.
- 별도 홍보 웹사이트와 iOS·Android 네이티브 앱은 초기 범위에서 제외한다.
- 데스크톱 브라우저에는 동일한 반응형 웹앱을 제공하며 별도 제품 경험으로 분리하지 않는다.
- 오프라인 캐시, 푸시 알림, 백그라운드 동기화는 구체적인 요구사항과 운영 정책이 확정될 때 별도로 결정한다.

## Consequences

- 하나의 Next.js 코드베이스와 배포 경로에 제품 개발을 집중할 수 있다.
- 설치 가능한 앱 형태와 일반 브라우저 접근이 같은 UI와 기능을 공유한다.
- 앱 스토어 전용 기능이나 네이티브 API가 필요해지면 별도 기술 결정과 범위 변경이 필요하다.
- 네트워크 단절 상태의 동작은 서비스 워커 전략을 결정하기 전까지 보장하지 않는다.
